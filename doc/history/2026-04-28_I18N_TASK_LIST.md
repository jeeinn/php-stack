## Phase 2：后端日志结构化 + 前端日志翻译

> **范围**: 后端 `ui_log!` 宏改造 + 前端日志接收翻译  
> **目标**: 后端发送结构化 key+args，前端用 vue-i18n 翻译后展示

### 2.1 后端改造 — 结构化日志事件

- [ ] **T2.1.1** 定义 `LogEvent` 结构体
  ```rust
  #[derive(Serialize)]
  pub struct LogEvent {
      pub key: String,
      pub args: serde_json::Value,  // 动态参数
      pub level: String,            // "info" | "warn" | "error"
  }
  ```
  - 位置: `src-tauri/src/logging.rs` 或新建 `src-tauri/src/log_event.rs`

- [ ] **T2.1.2** 改造 `ui_log!` 宏
  - 新签名: `ui_log!(app_handle, info, "log.key", { "param": value })`
  - emit payload 从纯文本改为 `LogEvent` JSON
  - 文件日志仍写英文纯文本（用 key 对应的英文默认值或直接写英文）

- [ ] **T2.1.3** 改造 `app_log!` 宏

> **范围**: Docker 原始输出、错误消息、日志导出  
> **目标**: 完善所有边界情况的 i18n 支持

### 3.1 Docker Compose 原始输出处理

- [ ] **T3.1.1** 定义 `log.raw` 特殊 key
  - 用于包装 docker compose stdout/stderr 的原始输出
  - 这些输出本身是英文，不需要翻译
  - 前端收到 `log.raw` key 时直接显示 args 中的 message，不做翻译

### 3.2 后端错误消息 i18n

- [ ] **T3.2.1** 梳理所有 `return Err(...)` 中的中文错误消息
  - `commands/env_config.rs`:
    - `"docker-compose.yml 文件不存在，请先应用配置"`
    - `"Docker Compose 启动失败"`
    - `"容器启动失败"`
    - `"Docker Compose 重启失败: {stderr}"`
    - `"Docker Compose 停止失败: {stderr}"`
  - `commands/workspace.rs`:
    - `"服务配置目录不存在: {}"`
    - `"指定的工作目录路径不存在"`
    - `"不支持的服务类型: {}"`
    - `"日志文件不存在，请先执行一些操作"`
  - `commands/mirror.rs`:
    - `"未知的镜像源类别: {}"`
  - `commands/backup.rs`:
    - `"所选路径不在项目根目录下..."`

- [ ] **T3.2.2** 选择错误消息 i18n 策略
  - **方案 A（推荐）**: 后端返回错误 key + args，前端翻译后展示
    ```rust
    return Err(serde_json::json!({
        "key": "error.compose_not_found",
        "args": {}
    }).to_string());
    ```
    前端 catch 时解析 JSON，用 `t()` 翻译
  - **方案 B**: 后端返回英文错误码，前端维护映射表
  - 需要改造前端所有 `catch (e)` 处的错误展示逻辑

| Phase | 涉及文件数 | 预估 i18n Key 数量 | 复杂度 |
|-------|-----------|-------------------|--------|
| Phase 1 | ~12 个 Vue 组件 + 2 个 composable | ~300-400 个 key | 中（量大但模式统一） |
| Phase 2 | ~5 个 Rust 文件 + 3 个前端文件 | ~80-100 个 log key | 高（需改造宏和事件协议） |
| Phase 3 | ~8 个 Rust 文件 + 前端错误处理 | ~30-40 个 error key | 中（边界情况多） |

## 附录：翻译文件 Key 命名规范

```
# 页面级
dashboard.*          # 环境管理页
envConfig.*          # 环境配置页
settings.*           # 设置页
mirror.*             # 镜像源面板
software.*           # 软件版本映射
migration.*          # 环境迁移页
backup.*             # 备份页
restore.*            # 恢复页
workspace.*          # 工作区初始化

# 通用
common.*             # 通用文本（取消、确认、删除、保存等）
error.*              # 错误消息

# 日志（Phase 2）
log.env.*            # 环境操作日志
log.backup.*         # 备份日志
log.restore.*        # 恢复日志
log.raw              # 原始输出（不翻译）

# 组件
component.select.*   # CustomSelect 组件
component.confirm.*  # ConfirmDialog 组件
```
