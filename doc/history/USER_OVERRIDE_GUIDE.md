# 用户版本覆盖功能使用指南

> **版本**: v0.1.0  
> **最后更新**: 2026-04-20

---

## 📋 功能概述

用户版本覆盖功能允许您自定义特定服务版本的 Docker 镜像标签，优先级高于默认配置。

**典型使用场景**：
- 🇨🇳 使用国内镜像源（阿里云、腾讯云等）
- 🔧 测试自定义构建的镜像
- 🚀 使用优化版本的镜像

---

## 🎯 使用方法

### 方法 1: 通过 UI 编辑（推荐）

#### 步骤 1: 打开软件设置页面

点击侧边栏的 **"🔧 软件设置"** 菜单项。

#### 步骤 2: 选择服务类型

在顶部标签中选择要修改的服务（PHP / MySQL / Redis / Nginx）。

#### 步骤 3: 编辑版本标签

1. 找到要修改的版本行
2. 点击 **"✏️ 编辑"** 按钮
3. 在弹出的对话框中输入新的 Docker 标签
4. （可选）添加备注说明
5. 点击 **"保存"**

**示例**：
```
版本号: 8.4
Docker 镜像标签: 8.4-lts-aliyun  ← 修改这里
备注: 使用阿里云镜像加速
```

#### 步骤 4: 重新应用配置

编辑完成后，需要重新应用环境配置才能生效：

1. 打开 **"环境配置"** 页面
2. 点击 **"应用配置"** 按钮
3. 等待配置生成完成
4. 重启容器使新镜像生效

#### 步骤 5: 验证生效

检查生成的 `.env` 文件：
```bash
cat .env | grep MYSQL84_VERSION
# 应该输出: MYSQL84_VERSION=8.4-lts-aliyun
```

### 方法 2: 手动编辑 JSON 文件（高级用户）

> 💡 **提示**：推荐使用方法 1（UI 编辑），更直观且不易出错。手动编辑仅适合高级用户。

#### 步骤 1: 创建/编辑配置文件

**文件位置**：项目根目录 `.user_version_overrides.json`（与 `.env`、`docker-compose.yml` 同级）

**文件格式**：
```json
{
  "_comment": "用户自定义的版本映射配置（优先级高于 version_manifest.json）",
  "_format": "{ service_type: { version: { tag: 'custom-tag', description: '备注' } } }",
  "mysql": {
    "8.4": {
      "tag": "8.4-lts",
      "description": "MySQL 8.4 LTS (使用默认配置)"
    }
  },
  "php": {},
  "redis": {},
  "nginx": {}
}
```

**字段说明**：
- `service_type`: 服务类型（`php`, `mysql`, `redis`, `nginx`）
- `version`: 版本号（如 `8.4`, `7.2`）
- `tag`: Docker 镜像标签（必需）
- `description`: 备注说明（可选）

#### 步骤 2: 重新应用配置

同上，需要在 UI 中重新应用配置。

---

## 🗑️ 删除自定义配置

### 方法 1: UI 删除

1. 在"软件设置"页面找到有自定义标记的版本
2. 点击 **"🗑️ 删除"** 按钮
3. 确认删除操作

### 方法 2: 重置所有

点击页面右上角的 **"🔄 重置所有自定义"** 按钮，清除所有自定义配置。

### 方法 3: 手动删除

删除项目根目录的 `.user_version_overrides.json` 文件即可。

## 📝 配置文件模板

如果您需要手动创建配置文件，可以参考以下完整模板：

```json
{
  "_comment": "用户自定义的版本映射配置（优先级高于 version_manifest.json）",
  "_format": "{ service_type: { version: { tag: 'custom-tag', description: '备注' } } }",
  "_example": "将 MySQL 8.4 改为使用特定标签",
  "mysql": {
    "8.4": {
      "tag": "8.4-lts",
      "description": "MySQL 8.4 LTS (使用默认配置)"
    }
  },
  "php": {},
  "redis": {},
  "nginx": {}
}
```

**注意事项**：
- ✅ 可以省略空的服务类型（如 `"php": {}`）
- ✅ `description` 字段是可选的
- ❌ 不要添加注释（JSON 不支持注释，这里的 `_comment` 仅用于说明）
- ❌ 确保没有尾随逗号

---

## ⚠️ 注意事项

### 1. 标签格式必须正确

❌ **错误示例**：
```json
{
  "tag": "latest"  // 不推荐，可能导致不可预期的行为
}
```

✅ **正确示例**：
```json
{
  "tag": "8.4-lts"  // 具体版本号
}
```

### 2. 镜像必须存在

确保您指定的标签在 Docker Hub 或配置的镜像源中存在，否则启动时会失败。

**验证方法**：
```bash
docker pull mysql:8.4-lts-aliyun
```

### 3. 重新应用配置

修改后**必须**重新应用配置才能生效：
- 编辑 `.user_version_overrides.json` → 不会自动生效
- 点击"应用配置" → 生成新的 `.env` 文件
- 重启容器 → 拉取新镜像

### 4. 备份自定义配置

建议定期备份 `.user_version_overrides.json` 文件，避免丢失。

## 🔍 故障排查

### 问题 1: 修改后未生效

**原因**：未重新应用配置

**解决**：
1. 打开"环境配置"页面
2. 点击"应用配置"
3. 检查 `.env` 文件是否更新

### 问题 2: 容器启动失败

**原因**：指定的镜像标签不存在

**解决**：
```bash
# 1. 检查镜像是否存在
docker pull mysql:your-custom-tag

# 2. 如果拉取失败，修正标签
# 3. 重新应用配置
```

### 问题 3: JSON 格式错误

**症状**：加载版本映射时显示错误

**解决**：
1. 检查项目根目录的 `.user_version_overrides.json` 格式
2. 使用 JSON 验证工具检查语法
3. 确保没有尾随逗号

**正确格式**：
```json
{
  "mysql": {
    "8.4": {
      "tag": "8.4-lts",
      "description": "描述"
    }
  }
}
```

## 📊 优先级规则

配置生效优先级（从高到低）：

1. **用户覆盖配置** (`.user_version_overrides.json`)
2. **默认版本清单** (`version_manifest.json`)
3. **用户输入回退** (直接使用版本号作为标签)

**示例流程**：
```
用户选择 MySQL 8.4
  ↓
检查 .user_version_overrides.json
  ├─ 有自定义配置 → 使用自定义 tag
  └─ 无自定义配置 → 检查 version_manifest.json
                      ├─ 有默认配置 → 使用默认 tag (8.4-lts)
                      └─ 无默认配置 → 使用用户输入 (8.4)
```

## 🎓 最佳实践

### ✅ 推荐做法

1. **添加备注说明** - 说明为什么使用自定义标签
2. **测试后再部署** - 先在开发环境测试
3. **记录变更** - 在团队中共享自定义配置
4. **定期清理** - 删除不再需要的自定义配置

### ❌ 避免的做法

1. **不要使用 `latest` 标签** - 导致不可预期的行为
2. **不要频繁修改** - 每次修改都需要重新拉取镜像
3. **不要忽略 EOL 警告** - 已停止维护的版本可能存在安全风险
