# V2.0 向导式环境搭建 - 测试指南

## 📋 功能概述

V2.0实现了向导式一键搭建开发环境，支持：
- ✅ PHP（自定义扩展安装）
- ✅ MySQL（固定密码 root123）
- ✅ Redis
- ✅ Nginx（基础配置）
- ✅ 镜像源配置（加速下载）

## 🚀 快速开始

### 1. 启动开发服务器

```bash
npm run tauri dev
```

### 2. 使用向导搭建环境

#### 步骤 1: 选择技术栈
- **必选服务**（默认勾选）：
  - PHP 8.2 ⭐
    - 扩展：mysqli, pdo_mysql, redis, gd, mbstring（默认勾选）
  - MySQL 8.0 ⭐
  - Redis 7.0 ⭐
  
- **可选服务**：
  - Nginx 1.24 ⭐（建议勾选，用于Web访问）

- **镜像源配置**：
  - APT: 阿里云（默认）
  - Composer: 阿里云（默认）
  - 点击"保存配置"按钮

#### 步骤 2: 预览配置
- 查看生成的 `docker-compose.yml` 内容
- 确认端口和依赖关系正确
- 点击"🚀 一键启动"

#### 步骤 3: 部署环境
- 观察实时日志输出
- 等待镜像构建和容器启动
- 成功后查看连接信息

## 🔍 验证部署

### 1. 检查容器状态

```bash
docker ps --filter "name=ps-"
```

应该看到以下容器：
- `ps-php-8-2`
- `ps-mysql-8-0`
- `ps-redis-7-0`
- `ps-nginx-1-24`（如果选择了Nginx）

### 2. 测试 MySQL 连接

```bash
docker exec -it ps-mysql-8-0 mysql -uroot -proot123 -e "SHOW DATABASES;"
```

应该看到 `app` 数据库。

### 3. 测试 Redis 连接

```bash
docker exec -it ps-redis-7-0 redis-cli ping
```

应该返回 `PONG`。

### 4. 测试 PHP 扩展

```bash
docker exec -it ps-php-8-2 php -m | grep -E "mysqli|pdo_mysql|redis|gd|mbstring"
```

应该列出所有已安装的扩展。

### 5. 测试 Nginx + PHP

创建测试文件：

```bash
mkdir -p data/www
echo "<?php phpinfo(); ?>" > data/www/index.php
```

访问 http://localhost，应该看到 PHP info 页面。

## 📁 目录结构

部署后的目录结构：

```
php-stack/
├── .env                    # 镜像源配置
├── data/
│   ├── mysql/             # MySQL 数据
│   ├── redis/             # Redis 数据
│   └── www/               # Web 根目录
├── nginx/
│   └── conf.d/            # Nginx 站点配置
├── build/
│   └── php-8.2-custom/    # PHP 自定义镜像构建目录
│       └── Dockerfile
└── docker-compose.yml     # 自动生成的配置文件
```

## 🐛 常见问题

### 1. 镜像构建失败

**症状**：日志显示网络超时或连接失败

**解决方案**：
1. 检查 `.env` 文件中的镜像源配置
2. 确保使用了国内镜像源（阿里云、清华等）
3. 重试部署

### 2. 端口冲突

**症状**：部署失败，提示端口已被占用

**解决方案**：
1. 停止占用端口的服务
2. 或在 Step 1 中修改端口映射

### 3. PHP 扩展未加载

**症状**：`php -m` 看不到已选择的扩展

**解决方案**：
1. 检查构建日志是否有错误
2. 删除 `build/php-8.2-custom` 目录重新构建
3. 确认扩展名称拼写正确

### 4. Nginx 无法访问 PHP

**症状**：访问 http://localhost 显示 404 或下载 PHP 文件

**解决方案**：
1. 确认 `data/www/index.php` 文件存在
2. 检查 Nginx 配置是否正确挂载
3. 重启 Nginx 容器：`docker restart ps-nginx-1-24`

1. 选择 PHP 7.4 + MySQL 5.7 + Redis 6.2
2. 扩展选择：mysqli, gd, curl
3. 部署并验证

### 测试无 Nginx 模式

1. 取消勾选 Nginx
2. 只部署 PHP + MySQL + Redis
3. 验证容器间网络通信

## 📊 性能基准

### 首次部署（无缓存）
- PHP 镜像构建：5-10 分钟（取决于扩展数量）
- MySQL/Redis/Nginx 拉取：1-2 分钟
- 总时间：约 10-15 分钟

### 二次部署（有缓存）
- 使用缓存镜像：秒级
- 总时间：约 1-2 分钟
