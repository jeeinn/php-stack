## 🎯 改进目标

借鉴 DNMP 项目的优秀实践，将 PHP-Stack 从**孤立容器管理**升级为**统一编排管理**，实现：

1. ✅ **统一网络**：所有容器共享 `php-stack-network`，可通过服务名互相访问
2. ✅ **集中配置**：动态生成 `docker-compose.yml`，统一管理所有服务
3. ✅ **智能重启**：新增/修改服务后，最小化重启受影响的容器
4. ✅ **服务发现**：PHP 可通过 `mysql:3306`、`redis:6379` 直接访问其他服务

## 📊 当前架构 vs 目标架构

### ❌ 当前架构（孤立容器）

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ ps-php-8-2   │     │ps-mysql-8-0  │     │ ps-redis-7-0 │
│              │     │              │     │              │
│ Network:     │     │ Network:     │     │ Network:     │
│ bridge (默认)│     │ bridge (默认)│     │ bridge (默认)│
└──────────────┘     └──────────────┘     └──────────────┘
      ↓                     ↓                     ↓
  无法通过服务名访问    无法通过服务名访问    无法通过服务名访问
  需要硬编码 IP       需要硬编码 IP       需要硬编码 IP
```

**问题：**
- PHP 连接 MySQL 需要知道具体 IP 地址
- 每次容器重启 IP 可能变化
- 配置文件需要手动更新

---

### ✅ 目标架构（统一网络 + Compose）

```
┌─────────────────────────────────────────────────┐
│          php-stack-network (自定义桥接网络)       │
│                                                   │
│  ┌──────────────┐  ┌──────────────┐             │
│  │ ps-nginx     │──│  ps-php-8-2  │             │
│  │ :80, :443    │  │  :9000       │             │
│  └──────────────┘  └──────┬───────┘             │
│                           │                      │
│                    ┌──────┴───────┐             │
│                    │ps-mysql-8-0  │             │
│                    │  :3306       │             │
│                    └──────┬───────┘             │
│                           │                      │
│                    ┌──────┴───────┐             │
│                    │ps-redis-7-0  │             │
│                    │  :6379       │             │
│                    └──────────────┘             │
└─────────────────────────────────────────────────┘

✅ PHP 可通过 "mysql:3306" 访问 MySQL
✅ PHP 可通过 "redis:6379" 访问 Redis
✅ Nginx 可通过 "php:9000" 访问 PHP-FPM
```

**优势：**
- 服务间通过**服务名**通信，无需关心 IP
- 容器重启后服务名不变
- 配置文件稳定可靠

## 🛠️ 实施方案

### 阶段 1：网络统一管理（立即实施）✅

#### 1.1 创建统一网络

在首次安装软件时自动创建 `php-stack-network`：

```rust
async fn ensure_network_exists(&self) -> Result<(), Error> {
    let network_name = "php-stack-network";
    
    // 检查网络是否存在
    match self.docker.inspect_network(network_name, None).await {
        Ok(_) => Ok(()), // 已存在
        Err(_) => {
            // 创建桥接网络
            self.docker.create_network(NetworkCreateRequest {
                name: network_name.to_string(),
                driver: "bridge".to_string(),
                check_duplicate: true,
                ..Default::default()
            }).await?;
            
            log::info!("✅ 创建统一网络: {}", network_name);
            Ok(())
        }
    }
}
```

#### 1.2 安装时自动加入网络

修改 `install_software` 方法，在容器启动后连接到统一网络：

```rust
pub async fn install_software(&self, spec: SoftwareSpec) -> Result<String, Error> {
    // ... 现有逻辑：创建并启动容器 ...
    
    // ✅ 新增：将容器加入统一网络
    self.connect_container_to_network(&container_name).await?;
    
    // ✅ 新增：更新 docker-compose.yml
    self.update_compose_file().await?;
    
    Ok(container_name)
}

async fn connect_container_to_network(&self, container_name: &str) -> Result<(), Error> {
    self.docker.connect_network(
        "php-stack-network",
        NetworkConnectRequest {
            container: container_name.to_string(),
            endpoint_config: Some(EndpointSettings {
                aliases: Some(vec![
                    // 添加短名称别名，便于服务发现
                    self.get_service_alias(container_name),
                ]),
                ..Default::default()
            }),
        }
    ).await?;
    
    log::info!("✅ 容器 {} 已加入 php-stack-network", container_name);
    Ok(())
}
```

**服务别名示例：**
- `ps-php-8-2` → 别名 `php`
- `ps-mysql-8-0` → 别名 `mysql`
- `ps-redis-7-0` → 别名 `redis`
- `ps-nginx-1-24` → 别名 `nginx`

### 阶段 2：动态生成 docker-compose.yml（核心功能）

#### 2.1 ComposeManager 实现

创建新文件：`src-tauri/src/engine/compose_manager.rs`

```rust
use serde::{Serialize, Deserialize};
use std::fs;
use std::path::Path;
use std::collections::HashMap;

/// Docker Compose 配置文件结构
#[derive(Debug, Serialize, Deserialize)]
pub struct DockerCompose {
    pub version: String,
    pub networks: HashMap<String, NetworkConfig>,
    pub services: HashMap<String, ServiceConfig>,
    pub volumes: Option<HashMap<String, VolumeConfig>>,
}

#[derive(Debug, Serialize, Deserialize)]
pub struct NetworkConfig {
    pub driver: String,
    pub external: Option<bool>,
}

#[derive(Debug, Serialize, Deserialize)]
pub struct ServiceConfig {
    pub image: String,
    pub container_name: String,
    pub networks: Vec<String>,
    pub ports: Option<Vec<String>>,
    pub volumes: Option<Vec<String>>,
    pub environment: Option<HashMap<String, String>>,
    pub depends_on: Option<Vec<String>>,
    pub restart: Option<String>,
    pub working_dir: Option<String>,
}

#[derive(Debug, Serialize, Deserialize)]
pub struct VolumeConfig {
    pub driver: String,
}

pub struct ComposeManager {
    compose_path: String,
}

impl ComposeManager {
    pub fn new(project_root: &str) -> Self {
        Self {
            compose_path: format!("{}/docker-compose.yml", project_root),
        }
    }

    /// 根据已安装的容器重建 docker-compose.yml
    pub async fn rebuild_from_containers(
        &self,
        containers: &[InstalledSoftware]
    ) -> Result<(), Box<dyn std::error::Error>> {
        let mut compose = DockerCompose {
            version: "3.8".to_string(),
            networks: HashMap::from([
                ("php-stack-network".to_string(), NetworkConfig {
                    driver: "bridge".to_string(),
                    external: Some(false),
                })
            ]),
            services: HashMap::new(),
            volumes: None,
        };

        // 为每个容器生成服务配置
        for container in containers {
            let service_name = self.extract_service_name(&container.name);
            let service_config = self.build_service_config(container)?;
            compose.services.insert(service_name, service_config);
        }

        // 序列化并写入文件
        let yaml = serde_yaml::to_string_pretty(&compose)?;
        fs::write(&self.compose_path, yaml)?;

        log::info!("✅ docker-compose.yml 已更新 ({} 个服务)", compose.services.len());
        Ok(())
    }

    /// 构建单个服务的配置
    fn build_service_config(
        &self,
        container: &InstalledSoftware
    ) -> Result<ServiceConfig, Box<dyn std::error::Error>> {
        let spec = &container.spec;
        
        // 端口映射: "宿主机端口:容器端口"
        let ports: Vec<String> = spec.port_mappings.iter()
            .map(|(container_port, host_port)| {
                format!("{}:{}", host_port, container_port)
            })
            .collect();

        // 数据卷挂载
        let volumes = if let Some(volume_path) = &spec.volume_path {
            if let Some(container_path) = spec.software_type.default_volume_path() {
                Some(vec![format!("{}:{}", volume_path, container_path)])
            } else {
                None
            }
        } else {
            None
        };

        // 依赖关系
        let depends_on = self.determine_dependencies(&spec.software_type);

        Ok(ServiceConfig {
            image: format!("{}:{}", 
                spec.software_type.default_image_prefix(),
                spec.version
            ),
            container_name: container.name.clone(),
            networks: vec!["php-stack-network".to_string()],
            ports: if ports.is_empty() { None } else { Some(ports) },
            volumes,
            environment: if spec.env_vars.is_empty() { 
                None 
            } else { 
                Some(spec.env_vars.clone()) 
            },
            depends_on,
            restart: Some("unless-stopped".to_string()),
            working_dir: None,
        })
    }

    /// 确定服务依赖关系
    fn determine_dependencies(
        &self,
        software_type: &SoftwareType
    ) -> Option<Vec<String>> {
        match software_type {
            SoftwareType::PHP => {
                // PHP 通常依赖数据库和缓存
                Some(vec![
                    "mysql".to_string(),
                    "redis".to_string(),
                ])
            }
            SoftwareType::Nginx => {
                // Nginx 依赖 PHP-FPM
                Some(vec!["php".to_string()])
            }
            _ => None,
        }
    }

    /// 从容器名提取服务名（去掉 ps- 前缀和版本号）
    fn extract_service_name(&self, container_name: &str) -> String {
        // ps-php-8-2 → php
        // ps-mysql-5-7 → mysql
        container_name
            .trim_start_matches("ps-")
            .split('-')
            .next()
            .unwrap_or(container_name)
            .to_string()
    }

    /// 执行 docker-compose up -d（应用变更）
    pub async fn apply_changes(&self) -> Result<(), Box<dyn std::error::Error>> {
        use tokio::process::Command;

        log::info!("🔄 应用 docker-compose 变更...");

        let output = Command::new("docker")
            .args(&[
                "compose",
                "-f", &self.compose_path,
                "up", "-d"
            ])
            .output()
            .await?;

        if !output.status.success() {
            return Err(format!(
                "docker compose 执行失败: {}",
                String::from_utf8_lossy(&output.stderr)
            ).into());
        }

        log::info!("✅ 服务已成功应用");
        Ok(())
    }

    /// 智能重启：只重启受影响的容器
    pub async fn smart_restart(
        &self,
        affected_services: &[String]
    ) -> Result<(), Box<dyn std::error::Error>> {
        use tokio::process::Command;

        for service in affected_services {
            log::info!("🔄 重启服务: {}", service);
            
            Command::new("docker")
                .args(&[
                    "compose",
                    "-f", &self.compose_path,
                    "restart", service
                ])
                .output()
                .await?;
        }

        Ok(())
    }
}
```

#### 2.2 集成到 SoftwareManager

修改 `software_manager.rs`：

```rust
use crate::engine::compose_manager::ComposeManager;

pub struct SoftwareManager {
    docker: Docker,
    compose_manager: ComposeManager,
}

impl SoftwareManager {
    pub fn new() -> Result<Self, Error> {
        let docker = Docker::connect_with_local_defaults()?;
        let compose_manager = ComposeManager::new("."); // 项目根目录
        
        Ok(Self { 
            docker, 
            compose_manager 
        })
    }

    pub async fn install_software(
        &self,
        spec: SoftwareSpec
    ) -> Result<String, Error> {
        // ... 创建并启动容器 ...
        
        // ✅ 更新 docker-compose.yml
        self.update_compose_after_install().await?;
        
        Ok(container_name)
    }

    pub async fn uninstall_software(
        &self,
        name: &str
    ) -> Result<(), Error> {
        // ... 停止并删除容器 ...
        
        // ✅ 更新 docker-compose.yml
        self.update_compose_after_uninstall(name).await?;
        
        Ok(())
    }

    /// 安装后更新 Compose 配置
    async fn update_compose_after_install(&self) -> Result<(), Error> {
        // 获取所有已安装的容器
        let containers = self.list_installed_software().await?;
        
        // 重建 docker-compose.yml
        self.compose_manager
            .rebuild_from_containers(&containers)
            .await?;
        
        // 应用变更（不会重启未变化的容器）
        self.compose_manager.apply_changes().await?;
        
        Ok(())
    }

    /// 卸载后更新 Compose 配置
    async fn update_compose_after_uninstall(
        &self,
        removed_name: &str
    ) -> Result<(), Error> {
        let containers = self.list_installed_software().await?;
        
        self.compose_manager
            .rebuild_from_containers(&containers)
            .await?;
        
        // 不需要重启，因为容器已删除
        Ok(())
    }
}
```

## ⚠️ 注意事项

### 1. 向后兼容性

- **已有容器**：需要手动迁移到统一网络
- **迁移脚本**：提供一键迁移工具

```rust
pub async fn migrate_existing_containers(&self) -> Result<(), Error> {
    let containers = self.list_installed_software().await?;
    
    for container in containers {
        self.connect_container_to_network(&container.name).await?;
    }
    
    self.update_compose_file().await?;
    
    log::info!("✅ 所有容器已迁移到统一网络");
    Ok(())
}
```

### 2. 端口冲突处理

- Compose 文件中端口冲突会导致启动失败
- 需要在生成配置前再次检测端口可用性

### 3. 数据卷持久化

- 确保数据卷路径正确映射
- 建议在项目根目录创建 `data/` 文件夹统一管理

```
project-root/
├── data/
│   ├── mysql/
│   ├── redis/
│   └── mongodb/
├── docker-compose.yml
└── ...
```
