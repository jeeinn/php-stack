## 📋 功能概述

软件管理中心是 PHP-Stack 的核心功能模块，提供多版本 PHP、MySQL、Redis、Nginx、MongoDB 等开发环境软件的一键安装与管理能力。

### 核心特性

- ✅ 支持 5 种软件类型：PHP (5.6-8.3)、MySQL (5.7-8.4)、Redis (6.2-7.2)、Nginx (1.24-1.25)、MongoDB (5.0-7.0)
- ✅ 智能端口自动分配，避免端口冲突
- ✅ 容器化参数配置（端口映射、数据卷、环境变量）
- ✅ 实时安装进度日志显示
- ✅ 一键安装/卸载，完整的生命周期管理

## 🏗️ 技术架构

### 后端架构 (Rust)

#### 核心模块：`src-tauri/src/engine/software_manager.rs`

**主要数据结构：**

```rust
// 软件类型枚举
pub enum SoftwareType {
    PHP, MySQL, Redis, Nginx, MongoDB
}

// 软件版本信息
pub struct SoftwareVersion {
    pub version: String,        // 版本号 "8.2"
    pub image_tag: String,      // Docker 镜像标签 "8.2-fpm"
    pub description: String,    // 描述信息
    pub is_stable: bool,        // 是否稳定版
}

// 软件安装规格
pub struct SoftwareSpec {
    pub software_type: SoftwareType,
    pub version: String,
    pub custom_image: Option<String>,       // 自定义镜像
    pub port_mappings: HashMap<u16, u16>,   // 端口映射
    pub volume_path: Option<String>,        // 数据卷路径
    pub env_vars: HashMap<String, String>,  // 环境变量
    pub extra_args: Vec<String>,            // 额外参数
}

// 已安装软件实例
pub struct InstalledSoftware {
    pub id: String,
    pub name: String,           // 容器名称 "ps-php-8-2"
    pub spec: SoftwareSpec,
    pub status: String,         // 运行状态
    pub created_at: String,     // 创建时间
}
```

**核心类：`SoftwareManager`**

主要方法：
- `get_available_versions()` - 获取可用版本清单
- `install_software(spec)` - 安装软件（异步）
- `uninstall_software(name)` - 卸载软件（异步）
- `list_installed_software()` - 列出已安装软件（异步）

**辅助类：`PortAllocator`**

- `is_port_available(port)` - 检测端口是否可用
- `find_available_port(start, end)` - 查找可用端口
- `allocate_ports(type, preferred)` - 自动分配端口

## 🔧 实现细节

### 1. 容器命名规范

所有容器遵循统一命名格式：`ps-{type}-{version}`

示例：
- `ps-php-8-2` (PHP 8.2)
- `ps-mysql-5-7` (MySQL 5.7)
- `ps-redis-7-0` (Redis 7.0)

版本号中的 `.` 被替换为 `-` 以符合 Docker 命名规范。

---

### 2. 端口分配策略

**默认端口映射：**

| 软件类型 | 容器端口 | 默认宿主机端口 |
|---------|---------|--------------|
| PHP     | 9000    | 9000         |
| MySQL   | 3306    | 3306         |
| Redis   | 6379    | 6379         |
| Nginx   | 80, 443 | 80, 443      |
| MongoDB | 27017   | 27017        |

**自动分配算法：**

```rust
1. 检查用户指定的首选端口是否可用
2. 如果可用，直接使用
3. 如果不可用，在 [preferred, preferred+100] 范围内查找第一个可用端口
4. 如果都不可用，回退到默认端口
```

使用 `TcpListener::bind()` 检测端口占用情况。

---

### 3. Docker 容器创建流程

```rust
install_software(spec) {
    1. 生成容器名称: ps-{type}-{version}
    2. 检查容器是否已存在
    3. 确定镜像名称 (自定义或默认)
    4. 拉取镜像 (如果本地不存在)
    5. 构建端口映射配置
    6. 构建数据卷挂载配置
    7. 合并环境变量 (默认 + 自定义)
    8. 创建容器 (bollard::create_container)
    9. 启动容器 (bollard::start_container)
    10. 返回容器名称
}
```

**关键技术点：**

- 使用 `bollard::container::Config` 构建容器配置
- 通过 `serde_json::json!` 宏序列化配置对象
- `HostConfig` 处理端口绑定和卷挂载
- 异步拉取镜像并显示进度日志

---

### 4. 数据卷持久化

对于有状态服务（MySQL、Redis、MongoDB），支持数据卷挂载：

```rust
// MySQL 示例
volume_path: "/path/to/host/mysql-data"
↓
容器内挂载到: /var/lib/mysql

// Redis 示例
volume_path: "/path/to/host/redis-data"
↓
容器内挂载到: /data
```

目前实现为可选配置，用户可以在 `SoftwareSpec` 中指定 `volume_path`。

---

### 5. 环境变量配置

不同软件类型的默认环境变量：

**MySQL:**
```env
MYSQL_ROOT_PASSWORD=root
MYSQL_DATABASE=default_db
```

**MongoDB:**
```env
MONGO_INITDB_ROOT_USERNAME=admin
MONGO_INITDB_ROOT_PASSWORD=admin
```

用户可以通过 `spec.env_vars` 覆盖或添加自定义变量。

   ```
   - 安装 PHP 7.4 → 验证容器 ps-php-7-4 创建
   - 安装 PHP 8.2 → 验证容器 ps-php-8-2 创建

   - 验证端口映射是否正确（9000, 9001）
   ```

2. **MySQL 数据卷持久化**
   ```
   - 安装 MySQL 5.7，指定数据卷路径
   - 创建测试数据库和表
   - 停止并删除容器
   - 重新安装 MySQL 5.7，挂载相同数据卷
   - 验证数据是否保留
   ```

3. **端口冲突处理**
   ```
   - 手动占用端口 3306
   - 安装 MySQL 5.7
   - 验证系统自动分配到 3307 或其他可用端口
   ```

4. **卸载功能**
   ```
   - 安装 Redis 7.0
   - 执行卸载操作
   - 验证容器已被删除
   - 验证端口已释放
   ```

## 📝 已知限制与改进方向

### 当前限制

1. **版本清单硬编码**
   - 当前版本列表在代码中硬编码
   - 未来可以从 Docker Hub API 动态拉取

2. **无版本缓存**
   - 每次切换类型都会重新加载
   - 可以添加内存缓存或本地存储

3. **错误处理较简单**
   - 网络错误、权限错误未区分
   - 可以增强错误分类和用户提示

4. **无批量操作**
   - 只能逐个安装/卸载
   - 可以添加"一键安装 LAMP 栈"功能

### 改进方向

#### 短期优化（v1.1.x）

- [ ] 添加版本缓存机制（TTL 1小时）
- [ ] 增强错误提示（网络超时、Docker 未启动等）
- [ ] 支持自定义 Docker 镜像标签
- [ ] 添加"推荐组合"快速安装（如 PHP 8.2 + MySQL 8.0 + Redis 7.0）

#### 中期规划（v1.2）

- [ ] 虚拟主机管理（Nginx 站点配置 GUI）
- [ ] SSL 证书自动生成（Let's Encrypt）
- [ ] 容器资源限制配置（CPU、内存）

#### 长期愿景（v2.0）

- [ ] 插件系统（支持扩展新软件类型）
- [ ] 云端同步（保存用户的软件配置到云端）
- [ ] 团队协作（共享环境配置模板）
